1. SOLID: is an acronym for five foundational design principles in OOP which helps Maintainability, Flexibility,  Testability, scalable, Reduced Complexity.
   1.1. S: Single Responsibility:  A class should have only one reason to change, focusing on a single task to improve cohesion
   1.2. O: Open/Closed: Software entities should be open for extension but closed for modification, allowing new functionality without altering existing code.
   1.3. L: Liskov Substitution: Subtypes must be substitutable for their base types without altering the correctness of the program.
   1.4. I: Interface Segregation: Clients should not depend on methods they do not use; prefer specific, small interfaces over a large, general one.
   1.5. D: Dependency Inversion:  High-level modules should not depend on low-level modules; both should depend on abstractions. Abstractions should not depend on details, but rather details should depend on abstractions.
